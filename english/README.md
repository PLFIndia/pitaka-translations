# Vendored English source (do not hand-edit)

`strings.xml` here is a **read-only mirror** of the canonical English source in
the app repo:

> `PLFIndia/pitaka` → `app/src/main/res/values/strings.xml`

It lives here only so the public CI in this repo can validate translations
without reaching into the private app repo. The app repo is the single source
of truth; this copy is kept in sync **automatically** by a workflow in the app
repo (`.github/workflows/sync-strings.yml`) that pushes here whenever the
canonical file changes. 

Do not edit this file by hand — your change would be overwritten on the next
sync, and worse, it would make the validator check translations against a
version of the English source that does not match what actually ships. If the
English copy looks stale, trigger the app-repo sync workflow rather than
editing here.

There is no developer secret stored in *this* (public) repo: the sync is a
one-way push *from* the private app repo, which holds the only credential.
